## 2. 项目背景

### 2.1 背景概述
新航道A level学员在课后刷题、做作业过程中，经常会遇到卡住的情况。大班课老师无法即时响应，而传统搜题App只能给答案无法互动讲解。即时解答模式旨在提供"一键呼叫、几分钟内匹配"的真人答疑服务，解决学习中的即时困难。

### 2.2 目标用户

**核心用户：新航道A level培训学员**

| 角色细分 | 痛点（表层） | 痛点（人性本质） | 核心诉求 | 现有解决方案 | 机会点 |
|---------|-------------|----------------|---------|-------------|--------|
| **即时答疑型** | 1. 做题卡住没人问<br>2. **作业马上要交，时间紧迫** | 1. **挫败感失控**——连续卡住摧毁自信<br>2. **时间流逝的恐慌**——每过一分钟焦虑倍增，deadline像倒计时炸弹<br>3. **侥幸心理后的崩溃**——以为能做出来，最后发现不行，时间所剩无几<br>4. **羞耻恐惧**——交不上作业/乱写被老师批评的难堪 | 1. 立刻摆脱被困状态<br>2. **在deadline前救命**<br>3. 快速搞懂不耽误交作业 | 1. 拍照搜题App（无互动，可能看不懂）<br>2. 抄同学答案（应付交差，心虚）<br>3. 乱写一通（知道会被打回来）<br>4. 熬夜硬撑（身心俱疲）<br>5. 不交作业（承担后果） | 1. **几分钟内匹配**——契合deadline紧迫感<br>2. **不仅给答案还快速讲懂**——下次类似题不再卡<br>3. **学习机内一键完成**——不用切换设备浪费时间<br>4. **新航道师资**——懂A level出题套路，讲得准 |

## 3. 产品目标

### 3.1 核心目标
1. 打造"5分钟内响应"的即时答疑体验
2. 通过AI+真人双模式覆盖不同难度和预算需求
3. 解决学生课后学习"无人可问"的痛点

### 3.2 成功指标

| 指标类型 | 具体指标 | 目标值 |
|---------|---------|--------|
| **服务质量** | 平均匹配等待时间 | ≤ 5分钟 |
| | 即时解答匹配成功率 | ≥ 90% |
| | 用户满意度 | ≥ 4.5/5 |
| **用户活跃** | 月活跃学生占比 | ≥ 60% |
| | 人均月发起咨询次数 | ≥ 3次 |
| **转化** | 注册到首次使用转化率 | ≥ 40% |
| | AI解答→在线解答转化率 | ≥ 20% |
| **收入** | 单用户月均消费 | ≥ ¥150 |
| | 在线解答完单率 | ≥ 85% |

## 4. 产品架构

### 4.1 服务模式

| 模式类型 | 核心定位 | 用户场景 | 时间特性 | 匹配机制 |
|---------|---------|---------|---------|---------|
| **AI文字解答** | 低价快速响应 | 基础概念、简单题目 | 秒级响应 | AI自动识别题目并生成解答 |
| **在线老师答疑** | 真人互动讲解 | 复杂问题、需要互动 | 5-10分钟内匹配<br>20-30分钟/次 | 一键呼叫，系统自动匹配 |

### 4.2 收费模式

| 服务类型 | 计费方式 | 价格策略 |
|---------|---------|---------|
| AI文字解答 | 按次收费 | 低价（如 ¥2-5/次） |
| 在线老师答疑 | 基础费+按时长计费 | 基础费¥X + ¥Y/分钟 |

### 5.1 学生端功能

#### 5.1.1 AI智能解答

**功能描述**：学生通过拍照或文字输入问题，AI自动识别并生成解答

**用户流程**：
```
选择AI解答 → 拍照/输入题目 → AI识别 → 展示解答 → 满意结束 / 不满意转在线
```

**核心功能点**：
- [ ] **拍照识题**：支持拍照上传题目图片，自动识别文字和公式
- [ ] **文字输入**：支持手动输入题目内容
- [ ] **AI解答生成**：基于题目生成详细解答步骤
- [ ] **相似题推荐**：推荐类似题目帮助巩固
- [ ] **转在线答疑**：AI解答不满意时可一键转真人老师

#### 5.1.2 在线老师答疑

**功能描述**：一键呼叫真人老师，进行视频/语音实时答疑

**用户流程**：
```
一键呼叫 → 选择学科/题型 → 排队等待 → 匹配成功 → 视频答疑 → 结束评价
```

**核心功能点**：
- [ ] **一键呼叫**：首页显著位置一键发起求助
- [ ] **学科选择**：选择问题所属学科（数学/物理/化学/经济等）
- [ ] **问题描述**：简短文字描述卡点或拍照上传
- [ ] **排队等待**：显示当前排队人数和预估等待时间
- [ ] **匹配成功通知**：震动/铃声提示，显示匹配老师信息
- [ ] **视频通话**：支持视频+音频通话
- [ ] **白板协作**：双向可写白板，支持手写公式、标注
- [ ] **屏幕共享**：学生可共享学习机屏幕展示题目
- [ ] **通话控制**：静音、切换前后摄像头、挂断
- [ ] **结束确认**：确认答疑结束，自动计算时长和费用
- [ ] **星级评价**：对老师进行1-5星评分和文字评价

#### 5.1.3 答疑历史

**功能描述**：查看过往所有答疑记录，支持回看视频

**核心功能点**：
- [ ] **历史列表**：按时间倒序展示历史答疑记录
- [ ] **筛选搜索**：按学科、日期筛选，关键词搜索
- [ ] **视频回放**：支持回看答疑过程录像
- [ ] **笔记收藏**：收藏重要题目的解答，添加个人笔记
- [ ] **错题本关联**：可标记为错题，关联到错题本

### 5.2 教师端功能

#### 5.2.1 接单中心

**功能描述**：接收和处理学生的即时答疑请求

**核心功能点**：
- [ ] **在线状态切换**：上线/离线/忙碌三种状态
- [ ] **接单/抢单**：系统自动派单或老师主动抢单
- [ ] **请求预览**：查看学生问题简述和学科
- [ ] **快速响应倒计时**：限时响应（如30秒）
- [ ] **通话工具**：视频/语音通话、白板、屏幕共享
- [ ] **课后备注**：记录学生问题类型，供后续参考

#### 5.2.2 个人中心

**核心功能点**：
- [ ] **资料编辑**：头像、昵称、简介
- [ ] **专长标签**：设置擅长学科和年级
- [ ] **在线时段设置**：设置通常可接单的时间段
- [ ] **收入统计**：今日/本周/本月接单数和收入

### 5.3 系统功能

#### 5.3.1 智能匹配

**功能描述**：根据学生问题自动匹配最合适的老师

**核心功能点**：
- [ ] **学科匹配**：根据学科标签匹配对应专长老师
- [ ] **难度评估**：根据题目内容评估难度等级
- [ ] **负载均衡**：优先分配给当前在线且等待较少的老师
- [ ] **超时转派**：一定时间内未匹配成功自动扩大匹配范围

#### 5.3.2 支付结算

**核心功能点**：
- [ ] **余额充值**：支持充值到账户余额
- [ ] **按次扣费**：AI解答按次扣费
- [ ] **按时长计费**：在线答疑基础费+时长费
- [ ] **教师结算**：服务完成后按规则结算给老师
- [ ] **账单明细**：消费记录和明细查询

## 6. 非功能需求

### 6.1 性能需求
- 页面加载时间 ≤ 3秒
- AI识题响应时间 ≤ 5秒
- 视频通话接通时间 ≤ 5秒
- 系统并发支持 ≥ 1000人同时在线

### 6.2 安全需求
- 通话内容录音留存，合规审核
- 学生/教师身份实名认证
- 支付安全（符合金融安全标准）
- 数据加密存储与传输

### 6.3 兼容性需求
- 适配主流学习机分辨率
- 支持横屏/竖屏切换
- 适配不同Android版本
